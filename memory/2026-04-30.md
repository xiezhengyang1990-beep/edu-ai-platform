- **根因发现**：上次改的 `.chart-wrapper` CSS 加了 `display:flex;align-items:flex-start`
  - 这是致命错误：canvas 作为 flex 子项，在 `align-items:flex-start` 下行内高度 = 0
  - 图表不是无限下滑，而是直接显示为空（高度0），Sam 看到的是空白的卡片
  - 之前的 `responsive: false` + 手动 canvas 尺寸是对的，但 CSS flex 覆盖了 canvas 的可视高度

#### ✅ 最终修复（第8次）
- `.chart-wrapper`：删除 `display:flex;align-items:flex-start`，恢复纯 `height:260px;overflow:hidden`
- `#revenueChart`：`display:block;width:100%`（只设宽不设高）
- Chart.js：恢复标准 `responsive:true + maintainAspectRatio:true + aspectRatio:3`
- 删除之前加的 `onResize` 手动回调（Chart.js 自带 ResizeObserver 处理）
- 删除之前加的 window resize listener（由 Chart.js 管理）
- **核心原理**：Chart.js 控制 canvas 实际高度（宽/3），wrapper `overflow:hidden` 防溢出，无循环
